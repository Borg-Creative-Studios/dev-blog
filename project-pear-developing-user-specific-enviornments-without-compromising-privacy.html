Project Pear: Developing User Specific Enviornments without Compromising Privacy

Borg Creative Studios Dev Team

We believe that privacy is a basic human right, but sadly many companies do not share this philosophy.

These days websites have extensive tracking and cookie-usage in order to create a “better” user experience. However, do these technologies actually improve the user’s experience in the long run? No.

By these companies tracking and recording so much information about their users, they run the risk of that data being hacked or obtained through other means. Which can eventually harm the user’s privacy and security.

But, how then does a company create a better user experience (like remembering a user’s page settings)? Well, that is where Project Pear comes into play.

Project Pear utilizes JavaScript and DOM to create user specific environments without any cookies or trackers. Users are able to create user names and edit the homepage links. By using JavaScript and DOM, the Project Pear website is able to store the user’s settings directly in their browser, not on a server. This not only helps protect the user’s privacy, but helps improve site load time by reducing the amount of data that has to be stored on the server.
